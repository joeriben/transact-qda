transact-qda — Installation


Prerequisites
-------------

  Docker — install for your platform:
      Linux:   https://docs.docker.com/engine/install/
      macOS:   https://www.docker.com/products/docker-desktop/
      Windows: https://www.docker.com/products/docker-desktop/
               (enable the WSL 2 backend during install)
     On macOS and Windows, launch Docker Desktop once after install
     and accept the system permission prompts. Wait until the whale
     icon in the menu bar / tray stops animating before continuing.
     The Docker Desktop window itself is not used to run transact-qda;
     it only needs to be running in the background. All commands below
     go into your terminal.
  Git


Install and start
-----------------

  1. Clone the repository:
         git clone https://github.com/joeriben/transact-qda.git
         cd transact-qda

  2. Set a session secret:
         echo "SESSION_SECRET=$(openssl rand -hex 32)" > .env

  3. Optional: pin a tested release tag:
         echo "TQDA_IMAGE=ghcr.io/joeriben/transact-qda:<tag>" >> .env

  4. Start:
         docker compose pull
         docker compose up -d

     The app image is pulled as a prebuilt multi-arch GHCR container.
     The first AI request downloads the embedding model
     nomic-embed-text-v1.5 (~150 MB). That happens only once.

To use a host port other than 5174, add TQDA_PORT=... to .env.


First run
---------

Open http://localhost:5174 in a browser.
Log in with username admin and password adminadmin.
Change the password at the yellow banner.


Everyday use
------------

    docker compose up -d       start
    docker compose down        stop (data is preserved in volumes)
    docker compose pull        fetch a newer published image
    docker compose logs -f     view logs
    ./scripts/4_db_migrate.sh  run migrations
    ./scripts/5_db_seed.sh     seed demo data
    ./scripts/6_db_backup.sh   create a database backup


Scope
-----

The supported runtime is the Compose stack above. Host-side Node helper
scripts are for development only and are not the installation path that
is promised to work the same way across macOS, Linux, and Windows.

For local image builds during development:

    docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d --build


Updating
--------

    git pull
    docker compose pull
    docker compose up -d
