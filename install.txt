transact-qda — Installation


Prerequisites
-------------

  PostgreSQL 16+
  Node.js 20+ and npm
  Git
  bash (native on Linux/macOS; on Windows via WSL2)


Linux
-----

  1. Install PostgreSQL 16, Node.js 20, and Git.

     Debian/Ubuntu:
         curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
         sudo apt-get install -y nodejs git postgresql postgresql-contrib

     Fedora:
         sudo dnf install nodejs npm git postgresql-server postgresql-contrib

  2. Create database and user:

         sudo -u postgres psql
         CREATE USER tqda WITH PASSWORD 'change-me';
         CREATE DATABASE transact_qda OWNER tqda;
         \q

  3. Clone and configure:
         git clone https://github.com/joeriben/transact-qda.git
         cd transact-qda
         cp .env.example .env

     Set DATABASE_URL and SESSION_SECRET in .env.

  4. Install and start:
         npm install
         ./scripts/4_db_migrate.sh
         ./scripts/3_start_prod.sh


macOS
-----

  1. Install PostgreSQL, Node.js, and Git via Homebrew:
         brew install postgresql@16 node git
         brew services start postgresql@16

  2. Create database and user:
         createuser -s tqda
         createdb transact_qda

  3. Clone and configure:
         git clone https://github.com/joeriben/transact-qda.git
         cd transact-qda
         cp .env.example .env

     Set DATABASE_URL and SESSION_SECRET in .env.

  4. Install and start:
         npm install
         ./scripts/4_db_migrate.sh
         ./scripts/3_start_prod.sh


Windows
-------

Run setup under WSL2.

  1. In PowerShell as administrator:
         wsl --install
     Restart when prompted. Set a username and password at first boot.

  2. Open the Ubuntu terminal from the Start menu. Continue from step 1
     of the Linux section.


First run
---------

Open http://localhost:5174 in a browser.
Log in with username admin and password adminadmin.
Change the password at the yellow banner.

The first AI request downloads the embedding model
nomic-ai/nomic-embed-text-v1.5 (~150 MB) to .model-cache/.


Start and stop
--------------

Start:
    ./scripts/3_start_prod.sh

Development:
    ./scripts/2_start_dev.sh

Stop:
    Ctrl+C in the terminal, or ./scripts/1_stop_all.sh


Updating
--------

    ./scripts/6_db_backup.sh
    ./scripts/1_stop_all.sh
    git pull
    npm install
    ./scripts/4_db_migrate.sh
    ./scripts/3_start_prod.sh


Parallel install on the same host
---------------------------------

    git clone https://github.com/joeriben/transact-qda.git transact-qda-next
    cd transact-qda-next
    cp .env.example .env

Use a separate `DATABASE_URL` and, if needed, a separate app port before
starting the second instance.
