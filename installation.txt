transact-qda — Installation


Zwei unterstützte Wege
----------------------

  1. PostgreSQL-basierte Installation für kleine Arbeitsgruppen
     Das ist der administrierte Weg. Er setzt einen Coding Agent oder
     eine Informatikerin / einen Informatiker im Haus voraus.

  2. Single-user-Installation auf einem einzelnen PC
     Das sollte über einen Installer laufen.


Weg 1: PostgreSQL für kleine Arbeitsgruppen
-------------------------------------------

Voraussetzungen:

  PostgreSQL 16+
  Node.js 20+ und npm
  Git
  bash (nativ unter Linux/macOS; unter Windows via WSL2)

Installieren und starten:

  1. Repository klonen und konfigurieren:
         git clone https://github.com/joeriben/transact-qda.git
         cd transact-qda
         cp .env.example .env

     In .env DATABASE_URL und SESSION_SECRET setzen.

  2. Abhängigkeiten installieren:
         npm install

  3. Starten:
         ./scripts/3_start_prod.sh

     Das Startskript wartet auf PostgreSQL, führt Migrationen aus und
     bootstrapped die initialen Admin-/Demo-Daten automatisch, falls die
     Datenbank noch leer ist.


Weg 2: Single-user-Installer
----------------------------

Für Linux- und macOS-Workstations existieren bereits erste Installer:

    sudo APP_PORT=5174 bash installer/install.sh
    bash installer/install_macos.sh

Diese Installer installieren PostgreSQL lokal, deployen die App, führen
Migrationen aus und bootstrappen die initialen Daten.
Zusätzlich legt er einen app-verwalteten lokalen PostgreSQL-Cluster im
State-Verzeichnis an, so dass der Nutzer PostgreSQL nicht manuell
administrieren muss.

Unter macOS läuft der entsprechende Weg benutzerlokal über Homebrew und
launchd statt über sudo + systemd.

Für eine breitere Desktop-Distribution für Endnutzer sollte dieser
Installer-Weg die Grundlage sein, nicht der Admin-Workflow oben.


Erster Start
------------

http://localhost:5174 im Browser öffnen.
Bei installer-verwalteten Linux-Setups geht alternativ auch:

    transact-qda-open

Bei installer-verwalteten macOS-Setups alternativ:

    ~/Applications/transact-qda-open.command

Mit Benutzername admin und Passwort adminadmin anmelden.
Passwort am gelben Banner ändern.

Der erste KI-Aufruf lädt das Embedding-Modell
nomic-ai/nomic-embed-text-v1.5 (ca. 150 MB) in den lokalen Model-Cache.
