transact-qda — Installation


Voraussetzungen
---------------

  Für die meisten Nutzer:innen ist dies der einfachste Weg:
      macOS / Windows: Docker Desktop
      Linux:           Docker Engine

  Docker — plattformspezifisch installieren:
      Linux:   https://docs.docker.com/engine/install/
      macOS:   entweder Docker Desktop
               (https://www.docker.com/products/docker-desktop/)
               oder, für Fortgeschrittene, Colima:
                   brew install docker colima
                   brew services start colima
      Windows: https://www.docker.com/products/docker-desktop/
               (bei der Installation WSL-2-Backend aktivieren)
     Unter macOS mit Docker Desktop: nach der Installation einmal
     starten, die System-Berechtigungsdialoge bestätigen und
     "Beim Anmelden starten" aktivieren. Warten, bis das Wal-Symbol
     in der Menüleiste nicht mehr animiert, bevor es weitergeht.
     Das Docker-Desktop-Fenster selbst wird nicht benutzt, um
     transact-qda zu starten; es stellt nur den Daemon im Hintergrund.
     Nach einem Mac-Neustart funktioniert transact-qda nur dann sofort
     weiter, wenn Docker Desktop ebenfalls wieder automatisch startet.
     Deshalb sollte "Beim Anmelden starten" aktiviert bleiben.
     Wer mit Begriffen wie Homebrew nichts anfangen
     kann, ignoriert Colima bitte und verwendet Docker Desktop.
     Unter macOS mit Colima aktiviert `brew services start colima`
     den Daemon samt Autostart, ohne dass Docker Desktop offen sein
     muss.
     Nach einem Mac-Neustart funktioniert transact-qda nur dann sofort
     weiter, wenn der Colima-Dienst wieder läuft.
     Wenn Docker Desktop und Colima parallel installiert sind, den
     Colima-Kontext einmal explizit setzen:
         docker context use colima
  Git

  Die folgenden Befehle werden in ein Terminal eingefügt:
      macOS:   Terminal
      Windows: Windows Terminal oder PowerShell
      Linux:   das normale Terminal


Installieren und starten
------------------------

  1. Terminal öffnen und in den Downloads-Ordner wechseln:
         cd ~/Downloads

  2. Projektdateien von GitHub herunterladen:
         git clone https://github.com/joeriben/transact-qda.git
         cd transact-qda

     Dadurch entsteht im Downloads-Ordner ein normaler Ordner namens
     `transact-qda`. Es wird nicht automatisch eine App in
     „Programme/Applications“ installiert.

  3. Session-Secret setzen:
         echo "SESSION_SECRET=$(openssl rand -hex 32)" > .env

  4. Optional auf getesteten Release-Tag pinnen:
         echo "TQDA_IMAGE=ghcr.io/joeriben/transact-qda:<tag>" >> .env

  5. Starten:
         docker compose pull
         docker compose up -d

     Das App-Image wird als vorgebautes Multi-Arch-GHCR-Image gezogen.
     Der erste KI-Aufruf lädt zusätzlich das Embedding-Modell
     nomic-embed-text-v1.5 (~150 MB). Das geschieht nur einmalig.

     Wenn `docker compose` meldet, dass der Docker-Daemon nicht
     erreichbar ist, läuft Docker Desktop / Colima noch nicht.

Für einen anderen Host-Port als 5174: TQDA_PORT=... in .env setzen.


Erster Start
------------

http://localhost:5174 im Browser öffnen.
Mit Benutzername admin und Passwort adminadmin anmelden.
Passwort am gelben Banner ändern.


Alltägliche Nutzung
-------------------

    docker compose up -d       starten
    docker compose down        stoppen (Daten bleiben in Volumes)
    docker compose pull        neues veröffentlichtes Image holen
    docker compose logs -f     Logs anzeigen
    ./scripts/4_db_migrate.sh  Migrationen ausführen
    ./scripts/5_db_seed.sh     Demo-Daten anlegen
    ./scripts/6_db_backup.sh   Datenbank sichern


Geltungsbereich
---------------

Der unterstützte Laufzeitpfad ist der obige Compose-Stack. Host-seitige
Node-Hilfsskripte dienen nur der Entwicklung und gehören nicht zu dem
plattformneutral zugesagten Installationspfad für macOS, Linux und
Windows.

Für lokale Image-Builds in der Entwicklung:

    docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d --build


Aktualisieren
-------------

    git pull
    docker compose pull
    docker compose up -d
