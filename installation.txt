transact-qda — Installation


Voraussetzungen
---------------

  Docker — plattformspezifisch installieren:
      Linux:   https://docs.docker.com/engine/install/
      macOS:   https://www.docker.com/products/docker-desktop/
      Windows: https://www.docker.com/products/docker-desktop/
               (bei der Installation WSL-2-Backend aktivieren)
     Unter macOS und Windows nach der Installation Docker Desktop
     einmal starten und die System-Berechtigungsdialoge bestätigen.
     Warten, bis das Wal-Symbol in der Menüleiste bzw. im Tray nicht
     mehr animiert, bevor es weitergeht.
     Das Docker-Desktop-Fenster selbst wird nicht benutzt, um
     transact-qda zu starten; es muss nur im Hintergrund laufen.
     Alle folgenden Befehle werden im Terminal eingegeben.
  Git


Installieren und starten
------------------------

  1. Repository klonen:
         git clone https://github.com/joeriben/transact-qda.git
         cd transact-qda

  2. Session-Secret setzen:
         cp .env.example .env
         # Fuer nicht-lokale Nutzung SESSION_SECRET zufaellig setzen

  3. Starten:
         docker compose up -d

     Der erste Start baut das Image (ca. 5 Minuten). Der erste
     KI-Aufruf lädt das Embedding-Modell nomic-embed-text-v1.5
     (~150 MB). Beides ist einmalig.

Für einen anderen Host-Port als 5174: TQDA_PORT=... in .env setzen.


Erster Start
------------

http://localhost:5174 im Browser öffnen.
Mit Benutzername admin und Passwort adminadmin anmelden.
Passwort am gelben Banner ändern.


Alltägliche Nutzung
-------------------

    docker compose up -d       starten
    docker compose down        stoppen (Daten bleiben in Volumes)
    docker compose logs -f     Logs anzeigen


Wartung
-------

    ./scripts/4_db_migrate.sh  offene Migrationen im App-Container ausfuehren
    ./scripts/5_db_seed.sh     Demo-Inhalte im App-Container anlegen
    ./scripts/6_db_backup.sh   Datenbank-Backup erzeugen


Aktualisieren
-------------

    git pull
    docker compose up -d
