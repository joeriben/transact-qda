     Plan: Document Viewer Naming List Interaction Redesign

     Context

     The document viewer's naming panel (column 2) currently uses single-click
     to code passages (create annotations). This conflates browsing/exploring
     namings with the coding action. The user wants a clearer separation:

     - Browsing = click/select namings → view their passages
     - Coding = drag a naming onto selected text

     Additionally: multi-selection of namings (shift/alt/drag), custom
     right-click context menu, and "Add to Cluster..." dialog.

     Bug to fix: After coding, the text selection state persists but the visual
     highlight disappears because DOM re-renders destroy the browser's native
     Range.
